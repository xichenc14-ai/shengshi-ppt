# 省心PPT 测试标准规范

> **版本：** v1.0
> **创建日期：** 2026-04-24
> **来源：** docs/TESTING-PLAN.md 核心内容同步

---

## 一、测试分级标准

| 等级 | 名称 | 触发时机 | 预计耗时 |
|------|------|---------|---------|
| P0 | 冒烟测试 | 每次发版 | <15分钟 |
| P1 | 回归测试 | 每周/重大更新 | <1小时 |
| P2 | 功能测试 | 新功能上线 | 按需 |
| P3 | 边界/性能 | 季度 | 按需 |

---

## 二、P0 冒烟测试（必测项）

每次发版必须通过的 7 项核心测试：

1. **首页访问** — 加载<3s，UI正确，无报错
2. **专业模式生成** — 完整流程：输入→大纲→编辑→生成→预览
3. **省心模式生成** — 输入→一键生成→结果展示
4. **附件上传** — PDF/Word 各一个，内容提取正确
5. **主题切换** — 7个色系展示，切换后预览更新
6. **预览功能** — 渲染正确，翻页正常
7. **导出功能** — PPTX/PDF 可下载可打开

**通过标准：** 7项全部通过 = P0 通过
**不通过处理：** 阻断发布，立即修复

---

## 三、缺陷等级

| 等级 | 定义 | 处理时效 |
|------|------|---------|
| S0 | 核心功能完全不可用 | 立即 |
| S1 | 主要功能异常 | 24h |
| S2 | 次要功能异常 | 3天 |
| S3 | UI/体验问题 | 下版本 |

---

## 四、测试验收流程

```
1. 开发完成 → 提交测试请求
2. 执行 P0 冒烟测试（<15分钟）
3. P0 通过 → 生成测试报告 → 可发布
4. P0 不通过 → 记录缺陷 → 阻断发布 → 修复后重测
5. P1 回归测试在发布后 24h 内完成
```

---

## 五、测试报告模板

每个测试报告必须包含：
- 测试时间/人员
- 测试环境（本地/预览/生产）
- 测试结果（逐项通过/失败）
- 发现的问题（含等级）
- 截图（失败项必须）
- 建议
