# 产品需求文档 (PRD) - Image to PPTX

**项目:** Image to PPTX  
**版本:** v0.1.0  
**日期:** 2026-05-22  
**状态:** 规划中

---

## 1. 项目概述与目标

### 1.1 项目背景

用户需要一个**本地 AI Agent 工具**，能够通过 AI 图像生成来驱动 PPT 制作流程。核心思路是将 AI 图像生成作为"素材中心"，而非直接生成 PPT。

### 1.2 项目目标

**核心目标：**
- 将用户提供的 PPT 需求和内容，通过 AI 生成"设计图"，再从设计图提取素材，最终用 Python 生成真实 PPTX 文件

**次要目标：**
- 支持无人值守运行
- 易于集成到现有 Agent 工作流
- 可作为 OpenClaw Skill 封装

### 1.3 关键价值

| 价值点 | 描述 |
|--------|------|
| **创意可视化** | 传统 PPT 制作需要手动设计，本工具通过 AI 生成整体设计图确保风格统一 |
| **效率提升** | 自动提取素材、自动布局，减少重复性工作 |
| **质量保证** | AI 生成的设计图经过精心策划，视觉呈现更专业 |
| **灵活定制** | 用户只需提供内容需求，设计细节由 AI 和工具自动处理 |

---

## 2. 用户需求分析

### 2.1 目标用户

- 需要快速制作 PPT 的职场人士
- 希望用 AI 辅助创意设计的人群
- 有自动化 PPT 生成需求的 Agent/开发者

### 2.2 用户痛点

1. **设计困难**：不知道如何配色、布局
2. **效率低下**：手动制作 PPT 耗时较长
3. **风格不统一**：手动调整容易出现风格不一致
4. **素材匮乏**：缺少合适的图标、图片素材

### 2.3 用户输入

用户只需提供：
- PPT 的主题/标题
- 内容要点/正文
- 期望的页数/篇幅（可选）
- 风格偏好（可选，如：商务、简约、活泼）

### 2.4 用户输出

用户获得：
- 一个完整的 `.pptx` 文件
- 可选的素材包（PNG 图标、无背景元素等）

## 3. 核心功能模块

### 3.1 模块列表

| 模块 | 英文名 | 职责 |
|------|--------|------|
| 内容分析器 | Content Analyzer | 理解用户需求，提取结构化信息 |
| 提示词工程师 | Prompt Engineer | 生成高质量的设计图生成提示词 |
| 图像生成器 | Image Generator | 调用 AI 生图 API 生成设计图 |
| 素材提取器 | Asset Extractor | 从设计图中提取视觉素材 |
| PPTX 构建器 | PPTX Builder | 使用 python-pptx 生成最终 PPTX |

### 3.2 模块间数据流

```
用户需求（文本）
    ↓
Content Analyzer → 结构化内容（主题、结构、要点）
    ↓
Prompt Engineer → 设计图提示词（prompt）
    ↓
Image Generator → PPT 设计图（PNG, 16:9）
    ↓
Asset Extractor → 素材包（图标、图片、背景）
    ↓
PPTX Builder → 最终 PPTX 文件
```

## 4. 输入/输出定义

### 4.1 输入定义

| 字段 | 类型 | 必填 | 描述 | 示例 |
|------|------|------|------|------|
| user_requirement | string | ✅ | 用户需求描述 | "制作一份关于Q3季度汇报的PPT，包含销售数据、市场分析和下季度计划" |
| output_path | string | ❌ | 输出路径 | "./output.pptx"（默认） |
| style | string | ❌ | 设计风格 | "modern" / "business" / "creative" |
| slide_count | int | ❌ | 幻灯片数量 | 10（自动推断） |
| model | string | ❌ | 图像生成模型 | "minimax"（默认） |

### 4.2 输出定义

| 字段 | 类型 | 描述 |
|------|------|------|
| pptx_path | string | 生成的 PPTX 文件路径 |
| assets | list | 素材包路径列表（可选） |
| design_image | string | 设计图路径（可选保留） |
| metadata | dict | 生成元数据（时间、模型消耗等） |

### 4.3 错误处理

| 错误类型 | 处理方式 |
|----------|----------|
| API 调用失败 | 重试 3 次，失败后返回错误信息 |
| 设计图质量不佳 | 提示用户可选择重新生成 |
| 素材提取失败 | 降级使用通用素材 |
| PPTX 生成失败 | 详细错误日志，保留中间产物 |

## 5. 验收标准

### 5.1 功能验收

- [ ] 用户提供自然语言需求，工具能生成对应 PPT
- [ ] 生成的 PPT 为有效的 `.pptx` 文件，可被 PowerPoint/WPS 打开
- [ ] PPT 内容与用户需求一致（主题、要点）
- [ ] 视觉风格统一、配色协调
- [ ] 支持 16:9 比例的 PPT 生成

### 5.2 性能验收

- [ ] 单次生成时间（从需求到 PPTX）不超过 5 分钟
- [ ] 支持无人值守运行，无需人工干预
- [ ] 错误处理完善，失败时有明确提示

### 5.3 集成验收

- [ ] 可通过 Python API 调用
- [ ] 可封装为 OpenClaw Skill
- [ ] 支持更换图像生成模型

### 5.4 Golden Sample 标准

生成的效果样本需满足：
1. 主题清晰：是关于"公司季度汇报"
2. 内容完整：包含封面、目录、数据页、总结页
3. 视觉专业：配色统一、布局合理
4. 可直接使用：无需进一步修改

## 6. 风险与挑战

### 6.1 技术风险

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| AI 设计图质量不稳定 | 中 | 提供多次生成选项，保留用户选择权 |
| 素材提取精度不足 | 中 | 降级方案：使用通用素材库 |
| 布局转换偏差 | 低 | 预留微调接口 |
| 中文字体兼容 | 中 | 使用系统常见中文字体，提供配置项 |

### 6.2 业务风险

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 用户需求理解偏差 | 中 | 设计图预览环节，让用户确认 |
| 生成内容版权问题 | 低 | 使用自有版权的 AI 模型 |

### 6.3 依赖风险

| 依赖项 | 风险 | 应对措施 |
|--------|------|----------|
| MiniMax API 可用性 | 低 | 预留其他模型接口 |
| python-pptx 稳定性 | 低 | 该库成熟度高 |
